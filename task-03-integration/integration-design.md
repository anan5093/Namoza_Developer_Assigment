Here is how we connect your landing page to HubSpot and Karix to trigger WhatsApp confirmations in under two minutes.

We use a Vercel serverless function as a middleware instead of HubSpot's native form or Zapier. Here is the catch: HubSpot deduplicates records using email address by default. This is like a post office filing mail by house address—if two different tenants live in the same building, their letters get mixed up. Since we only collect name and phone, standard forms will merge different patients sharing a phone number into a single messy contact. Our serverless function fixes this. When a patient submits, the function queries HubSpot's Contact Search API by phone number. If we find a match, we check the name. If it matches, we update the existing record's lead status to 'New Enquiry'. If it is a different name—like a mother booking for her child using her phone—we create a new contact so we do not overwrite the mother's clinical history.

Once the contact is processed, the function calls the Karix API to send the WhatsApp confirmation. Google Ads conversion tracking fires directly on the user's browser right before this background work starts.

The weakest link is the Karix API. If Karix goes down, patients do not get their confirmation. To prevent this, we do not call HubSpot and Karix directly from the frontend. The serverless function acts as a quick mailbox, dropping the form data into an AWS SQS queue and immediately showing the thank-you screen. A background worker then pulls from this queue to update HubSpot and trigger Karix. If Karix fails, the worker retries the job automatically using exponential backoff.

Three things can break our two-minute WhatsApp SLA: Karix API rate limits during peak ad spend, a backup in our queue processing, or HubSpot API rate limits. If you get stuck here, it is usually because of queue backlog. We monitor this by setting up CloudWatch alerts on our queue's message age. If any message sits for more than 90 seconds, it pages our on-call team. We also monitor Karix delivery status webhooks; if a message status does not show 'Delivered' within two minutes, we flag it in our Slack dashboard so we can call the patient manually.
