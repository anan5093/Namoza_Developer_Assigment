Here is how I would connect this landing page to HubSpot, WhatsApp, and Google Ads.

Instead of using a native HubSpot embed—which breaks our custom landing page styling—or adding Zapier, which adds unnecessary overhead for transferring one form payload, I would write a Vercel serverless function. But here is the catch: HubSpot automatically deduplicates contacts by email. This is like a post office filing mail by house address—if two different tenants live in the same building, their letters get mixed up. Since this form only collects name and phone, a standard integration would overwrite the existing contact's name every time a new patient submits with the same phone. Our serverless function resolves this. It queries HubSpot’s Contact Search API by phone number. If the name matches, it updates the contact. If the name is different—like a mother booking for her child using her phone—it creates a new contact to preserve clinical history.

Once the contact is updated or created, the function calls the Karix API to send the WhatsApp confirmation. Google Ads conversion tracking fires on the user's browser right before this.

The weakest link is the Karix API connection. If Karix goes down, the WhatsApp SLA breaks. To prevent this, the serverless function acts as a quick mailbox, dropping form data into an AWS SQS queue and immediately showing the thank-you screen. A background worker then pulls from this queue to update HubSpot and trigger Karix. If Karix fails, the worker retries the job using exponential backoff.

Three failure points could break our two-minute WhatsApp SLA: Karix rate limits, queue backlog, or HubSpot API limits. If you get stuck here, it is usually queue backlog. I would monitor this by comparing the creation timestamp against the message sent timestamp in our queue. If this difference exceeds 90 seconds, we trigger a CloudWatch alert to our team. We also monitor Karix webhooks; if a message status does not show 'Delivered' within two minutes, we flag it in Slack so we can call the patient manually.
